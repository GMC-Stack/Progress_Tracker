## How Can I Contribute?

There are several ways to contribute, regardless of your technical background:

- 🐛 **Report a bug** — something is broken or not working as expected
- 💡 **Suggest a feature** — you have an idea to improve the tracker
- 📝 **Improve documentation** — fix a typo, clarify an explanation, translate content
- 🔧 **Submit a fix or improvement** — directly improve the file or code
- 🌍 **Translate** — help make the tracker available in other languages

---

## Reporting Bugs

If you find a bug, please open an **Issue** on GitHub and include:

1. A clear and descriptive title
2. Steps to reproduce the problem
3. What you expected to happen
4. What actually happened
5. Your Excel version and operating system
6. Screenshots if helpful

> 💡 Before opening a new issue, please check if a similar one already exists.

---

## Suggesting Enhancements

Have an idea for a new feature or improvement? Open an **Issue** and include:

1. A clear description of the enhancement
2. Why you think it would be useful
3. Any examples or mockups if you have them

Please be as specific as possible — the more detail you provide, the easier it is to evaluate and implement your suggestion.

---

## Submitting Changes

If you want to directly contribute a fix or improvement:

1. **Fork** this repository
2. **Create a new branch** with a descriptive name  
   *(e.g. `fix-formula-wellbeing` or `add-new-dimension`)*
3. **Make your changes**
4. **Test** that everything still works correctly
5. **Submit a Pull Request** with a clear description of what you changed and why

### For Excel file changes:
- Do not break existing formulas or references
- Keep the visual style consistent with the existing sheets
- Test with at least 4 weeks of sample data before submitting

### For code changes (React tracker):
- Keep the code readable and well-commented
- Test in at least one modern browser before submitting

## Style Guidelines

To keep the project consistent:

- **Language:** English for all documentation and code comments
- **Tone:** Clear, friendly, and professional
- **Excel:** Times New Roman font, consistent color palette (blues and greens)
- **Code:** Clean and readable — prioritize clarity over cleverness

---

## Code of Conduct

This project follows a simple principle:

> *Be respectful, constructive, and kind.*

All contributions are welcome regardless of experience level.  
Criticism should always be directed at ideas, never at people.
